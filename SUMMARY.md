## Segment 5 â€” Housekeeping: Rename, coreai Cleanup & Beads Onboarding

**Topic:** Project folder rename, stale reference cleanup, and new developer onboarding for Beads

**Key Decisions:**

- `coreai` references fall into two categories: (1) actionable â€” wrong project name in config/lock files; (2) intentional â€” Beads issue IDs (`coreai-*`) are live database identifiers tied to the Dolt DB name; SonarCloud project key `jwill9999_coreai` is an external identifier that cannot be renamed without recreating the project

**Constraints:**

- Historical CHANGELOG entries and ADR-0003 mentioning `@coreai` are correct â€” they document the rename event

**Outcome:**

## Segment 6 â€” Codecov Integration Debugging (Unresolved)

**Topic:** Diagnosing and attempting to fix "Missing Head Report" on Codecov for main branch

**Key Decisions:**

- `--no-cache` in Nx only prevents writing to cache, not reading â€” use `--skip-nx-cache` to force tests to run fresh and write coverage files
- Nx Cloud remote cache shares results between branch CI and main CI â€” tests served from cache in 2s on main because the same code was already tested on the branch
- `agent-types` is a pure type-definitions package; Jest produces a 0-byte lcov.info for it â€” removed from Codecov upload to avoid poisoning the batch
- Jest generates lcov `SF:` paths relative to each package dir (e.g. `src/utils.ts`) not the repo root â€” added a `sed` step to prefix them with the package path before upload
- Diagnostic approach agreed: test Codecov on a PR branch first to isolate whether the issue is main-specific or a fundamental config problem

**Constraints:**

- The changelog bot always creates a `[skip ci]` commit on top of every push to main â€” this commit becomes HEAD with no CI run and no coverage report
- `carryforward: true` in `codecov.yml` only helps if there's a prior successful upload to carry forward from

**Outcome:**

- 3 CI commits pushed to main (1be62fc, 0d4a0ee, cf9e1b2); all CI runs pass but Codecov still shows "Missing Head Report"
- Root cause unconfirmed â€” none of the attempted fixes resolved it
